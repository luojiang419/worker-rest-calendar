# 工作日历 · Codex 项目蓝图包 v1

这是一个供 Codex 按阶段实施的完整项目包，目标是开发一款支持双休、单休、大小周、循环班制与临时调休的个人休息日历。

## 产品一句话

让用户无需心算排班，打开手机或桌面挂件即可知道：今天是否上班、距离下次休息还有几天、本周属于大周还是小周。

## 明确的大小周定义

- **大周**：周六、周日双休。
- **小周**：周六上班，周日休息。
- 大周与小周按周交替；用户必须指定一个“锚点周”，例如“2026-07-06 所在周是大周”。

## 推荐技术路线

- Flutter + Dart，共用移动端和桌面端的大部分业务与 UI 代码。
- iOS / Android：完整应用；原生桌面小组件由 Flutter 与平台原生代码桥接。
- macOS / Windows：完整应用 + 无边框、可拖拽、可置顶的桌面挂件窗口。
- 本地优先：SQLite/Drift；可选 Supabase 同步放在后期阶段。
- 状态管理：Riverpod；路由：go_router。

依赖版本不在蓝图中硬编码。Codex 应在 Phase 00 根据当前 Flutter stable 与目标平台解析兼容版本并生成 lockfile。

## 交给 Codex 的使用方法

1. 将本项目包解压到仓库根目录。
2. 让 Codex 先阅读 `AGENTS.md` 和 `codex/MASTER_PROMPT.md`。
3. 依次执行 `codex/PHASE_00` 到 `PHASE_10`，不得跳过排班引擎测试阶段。
4. 每完成一个阶段，更新 `PROGRESS.md`、运行格式化、静态检查和测试。
5. UI 以 `docs/07_UI_UX_SPEC.md`、`config/design_tokens.json` 为准，图片只作为视觉方向参考。

## 目录说明

- `docs/`：产品、功能、排班算法、架构、数据、UI、测试、发布说明。
- `codex/`：可直接逐阶段交给 Codex 的实施任务。
- `ui/`：完整视觉图、局部参考、页面目录、组件目录、可浏览的静态样式板。
- `config/`：机器可读设计令牌、示例排班、功能开关和状态定义。
- `starter/`：Flutter 起始结构和排班引擎示例代码。
- `scripts/`：项目包自检脚本。

## MVP 完成标准

用户可完成首次班制设置，准确显示未来日期的上班/休息状态，支持单日覆盖、调休、请假和加班备注；手机端具有浅色/暗黑模式；桌面端具有小/中/大三种挂件；提醒、统计、导入导出和核心测试全部通过。
