# Phase 11 · 精致桌面摆件套件

## 目标

在现有单窗口桌面摆件架构中新增可切换的精致时钟、快速记事本和专注计时器，并与排班摆件共享五种视觉风格、浅色/暗黑模式、尺寸、透明度、置顶、锁定、托盘和跨显示器定位能力。

## 范围约定

- 本阶段只维护一个无边框 Flutter 窗口，通过设置页或右键菜单切换摆件类型。
- 不引入多窗口、多进程、天气 API、账号或新的网络权限。
- 第一批类型固定为：排班、时钟、记事本、专注计时器。
- 记事本为一个快速便笺，最多 4000 字，本地自动保存并进入备份导入导出。
- 专注计时器默认 25 分钟，运行时依据目标结束时间计算，系统睡眠恢复后自动校正。

## 模块

1. 类型与持久化：`DesktopWidgetType`、数据库 v8、设置仓储和备份兼容。
2. 公共表面：复用设计令牌的无描边卡片、阴影安全边距和拖动手柄。
3. 时钟：时间、秒、日期、星期、今日排班与下一休息日。
4. 记事本：多行编辑、500ms 防抖自动保存、字符限制和拖动手柄。
5. 专注计时器：开始、暂停、继续、重置、真实剩余时间和完成状态。
6. 入口：设置页与 Windows 原生右键菜单切换类型。

## 验收标准

- 四种类型切换后立即刷新，重启应用仍保持最后选择。
- 五种视觉风格在浅色和暗黑模式下均从设计令牌取值，无硬编码表面颜色、圆角和阴影。
- 小、中、大尺寸及 130% 字体下无核心溢出。
- 时钟跨分钟、跨午夜和前台恢复后显示正确。
- 记事内容自动保存，重启、导出、清空、导入后可恢复。
- 专注计时器暂停不丢失剩余时间，睡眠/阻塞后按目标结束时间校正。
- 记事编辑不触发窗口拖动；拖动手柄在未锁定时可移动窗口。
- 原有排班摆件、托盘、透明度、置顶、锁定与跨分辨率定位测试保持通过。
- 格式化、静态分析、单元/Widget 测试和 Windows 集成测试通过后才允许更新 `PROGRESS.md`。
