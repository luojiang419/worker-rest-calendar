# 代码变更备份 001 · Phase 00–03

本文件记录蓝图初始包到 Phase 03 完成时的主要源码差异，不复制构建缓存或用户数据。

## 工程底座

- 修改前：根目录无正式 Flutter 工程，仅有 `starter/`。
- 修改后：生成 `android/`、`ios/`、`macos/`、`windows/`、`lib/`、`test/`、`.github/workflows/flutter.yml` 和锁文件。
- 依赖基线：Flutter 3.38.8、Dart 3.10.7、Riverpod 3.3.1、Drift 2.31.0、SQLite 2.9.4。

## 排班领域

- 修改前：starter 中仅有简化 `DateTime` 规则和少量大小周测试。
- 修改后：新增 `lib/core/date/` 与 `lib/features/schedule/domain/`，使用 `CalendarDate`、数学 floor division、正规范化取模和不可变覆盖映射。
- 新增行为：六种班制、覆盖解析、下一休息日、计划/实际连续工作、诊断错误。

## 数据层

- 修改前：无 SQLite 表、仓储或导入导出实现。
- 修改后：新增 `lib/core/database/`、schedule/settings/sync repositories、schema v2 生成代码和迁移备份器。
- 事务行为：active profile 唯一切换、覆盖编辑、导入整批回滚、同日期 UUID 冲突替换。

## 设计系统

- 修改前：`AppTheme` 仅含少量硬编码背景和种子色，首页为工程占位文字。
- 修改后：新增 `AppTokens ThemeExtension`，所有基础组件从令牌读取颜色、阴影、圆角、间距和动效。
- 新增组件：卡片、主/次/危险按钮、分段控件、五状态标签、日期胶囊、底部弹层、Toast、加载/空/错状态。
- 当前路由：`/` 指向 `ComponentGalleryPage`，Phase 04 将替换为首次启动入口。
