# 进度快照 003 · Phase 05 首页、日历与日期编辑完成

记录时间：2026-07-13（Asia/Shanghai）

## 已完成内容

- 建立唯一共享 `ActiveScheduleState`，将 active profile、手动覆盖和节假日覆盖组装为 `ScheduleEngine`。
- 建立 `DayPresentation`，首页、月历、周历、详情和编辑均消费同一解析结果，不在组件内重复计算排班。
- 今日首页完成四张核心卡：今日状态、下次休息、本周七日、本月概览。
- 大小周首页显示“大周/小周”文字标签；五种日期状态均有文字、图标和语义。
- 月视图固定展示从周一开始的 42 天；周视图展示周一至周日 7 天。
- 支持月/周分段切换、前后周期按钮、左右滑动、回到今天和跨年月份切换。
- 日期点击打开详情底部面板，长按直接进入编辑；首页“修改今天”和下次休息入口已接通。
- 编辑支持工作、休息、调休上班、调休休息、请假、加班分钟和备注。
- 已有覆盖编辑时复用 UUID/创建时间；删除使用软删除并恢复节假日或基础班制结果。
- 每次实际保存/删除后重载共享状态，并发布 `ScheduleRefreshEvent` 供后续提醒/小组件订阅。
- active profile 冷启动改用一次性查询，避免一次性页面读取依赖 Drift watch 首次发射。
- 新增 12 项测试，全量测试从 63 项增加至 75 项。

## 关键代码前后对比

### 首页数据源

修改前：

```dart
class HomePage extends StatelessWidget {
  // 仅展示“首次设置已完成”的占位卡片
}
```

修改后：

```dart
final activeScheduleControllerProvider =
    AsyncNotifierProvider<ActiveScheduleController, ActiveScheduleState>(...);

final dashboard = TodayDashboard.build(state, currentDate);
```

首页与日历均读取 `activeScheduleControllerProvider`，同一天只存在一个有效解析结果。

### active profile 冷启动读取

修改前：

```dart
final profile = await repository.watchActiveProfile().first;
```

修改后：

```dart
final profile = await repository.getActiveProfile();
```

持续监听接口仍保留，但一次性冷启动使用直接 Drift 查询，解决 Widget 帧环境中页面长期 loading 的问题。

### 单日覆盖刷新

修改前：数据仓储支持保存/软删除，但应用 UI 没有编排和刷新入口。

修改后：

```dart
await repository.saveDayOverride(override);
state = AsyncData(await _load());
publish(ScheduleRefreshEvent(date, saveOverride));
```

保存/删除先落本地，再刷新共享 provider，最后发布带日期的后续刷新事件。

### 日历日期格

修改前：没有月/周业务日历。

修改后：`CalendarDayCell` 显示日期数字和“班/休/调班/调休/假”，并为今天、手动调整、选中态提供语义；小屏 130% 字体使用叠层今天标记与按文字缩放的行高，避免溢出。
