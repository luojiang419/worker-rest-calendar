## 关键代码前后对比

### 应用根入口

修改前：

```dart
GoRoute(
  path: '/',
  builder: (context, state) => const ComponentGalleryPage(),
)
```

修改后：

```dart
GoRoute(
  path: '/',
  builder: (context, state) => const AppStartupPage(),
)
```

`AppStartupPage` 读取本地首次启动状态：未完成时进入首次设置，完成后进入今日占位首页。

### 首次设置状态

修改前：没有首次设置领域模型或中断恢复能力。

修改后：

```dart
final class OnboardingDraft {
  final OnboardingStep step;
  final SchedulePatternType? patternType;
  final CalendarDate? anchorDate;
  final WeekType? anchorWeekType;
  final List<DayKind> cycleDays;
}
```

草稿以 `onboarding_draft_v1` 键保存到 SharedPreferences，并对损坏 JSON 做安全清理。

### 完成设置

修改前：数据库尚未接入应用启动流程，也不会创建 active profile。

修改后：控制器校验草稿后创建 `ScheduleProfile(isActive: true)`，保存 `firstLaunchCompleted: true`，清除草稿并切换到完成状态；重复点击完成由应用层防重入保护。

### 大小周预览

修改前：只有 Phase 01 排班引擎测试，没有首次设置预览界面。

修改后：预览复用 `AlternatingBigSmallWeekPattern`，生成未来 30 天逐日结果；另生成 56 天结果并按 8 周展示，测试逐周断言大/小周与周六状态。

### 分段控件空选择

修改前：`AppSegmentedControl` 默认要求至少一个已选项，大小周首次进入未选择周类型时会触发 Flutter 断言。

修改后：增加默认关闭的 `emptySelectionAllowed` 参数，仅大小周配置页显式开启，不改变其他现有使用方行为。
