# 020 · 精致桌面摆件云端发布前

## 当前目标

在原有单窗口 Windows 桌面摆件中新增时钟、记事本和专注计时器，与排班摆件共享主题、窗口能力和发布链路；完成测试后推送 `main`，由公开 GitHub 仓库自动生成正式 Release。

## 已完成内容

1. 新增 `DesktopWidgetType`：排班、时钟、记事本、专注计时。
2. 设置页和 Windows 原生托盘菜单均可切换摆件类型，选择立即生效并持久化。
3. 数据库从 v7 升级到 v8，保存摆件类型和最多 4000 字的便笺；旧数据库和旧备份默认回退到排班与空便笺。
4. 新增共享 `DesktopWidgetFrame`，四种摆件统一使用设计令牌、无描边表面、透明阴影安全边距和小/中/大尺寸内边距。
5. 时钟每秒显示真实本地时间，并显示日期、星期、今日排班和下次休息日；跨午夜和前台恢复会校正。
6. 记事本使用独立拖动手柄，正文编辑不会拖动窗口；500ms 防抖自动保存，销毁前补写最后内容，锁定位置后仍可编辑。
7. 专注计时器支持 25 分钟开始、暂停、继续、重置和完成状态；使用目标结束时间计算，系统睡眠或阻塞后不会按丢失的 tick 漂移。
8. 记事本和专注计时不再订阅排班数据，启动时不会被排班加载状态阻塞。
9. 新增五种视觉风格 × 明暗主题组合测试，以及小/中/大尺寸和 130% 字体回归。

## 关键代码前后对比

### 摆件路由

修改前：`DesktopWidgetShell` 始终读取 `activeScheduleControllerProvider`，只构建 `DesktopWidgetCard`。

修改后：根据 `preferences.desktopWidgetType` 路由到排班、`DesktopClockCard`、`DesktopNoteCard` 或 `DesktopFocusCard`；排班订阅被收敛到 `_ScheduleBackedWidget`，非排班摆件独立加载。

### 设置模型与数据库

修改前：`AppPreferences` 与 `AppSettings` 不记录摆件内容类型，数据库 schemaVersion 为 7。

修改后：增加 `desktopWidgetType`、`desktopWidgetNote`，数据库 schemaVersion 为 8；迁移为旧库添加默认 `schedule` 和空字符串列，Drift 生成文件已更新。

### 专注计时

修改前：无专注模块。

修改后：`FocusTimerController` 保存目标结束时间，在 `synchronize()` 中以 `targetEnd.difference(now)` 重新计算剩余秒数；卡片使用确定性圆形进度、明确剩余时间和状态文字。
