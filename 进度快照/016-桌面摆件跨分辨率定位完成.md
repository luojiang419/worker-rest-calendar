# 进度快照 016 · 桌面摆件跨分辨率定位完成

## 已完成内容

### 1. 自适应定位算法

- 新增纯 Dart `DesktopWindowPlacement` 模型。
- 距离屏幕边缘不超过 96 逻辑像素时保存边缘锚点和边距。
- 中间区域保存窗口可移动范围内的横纵比例。
- 恢复时基于当前显示器可用工作区和当前摆件尺寸重新计算，并限制到可见范围。

### 2. V2 位置存储与迁移

- 新增 `desktop_widget_placements_v2` 和 `desktop_widget_last_display_v2`。
- 每个显示器分别保存水平、垂直锚点和数值。
- V1 `dx/dy` 首次读取时结合旧显示器工作区自动迁移。
- 损坏 JSON、非法比例和无效旧坐标均安全回退，不阻塞应用启动。

### 3. Windows 显示变化监听

- Windows runner 监听 `WM_DISPLAYCHANGE`、`WM_DPICHANGED` 和 `SPI_SETWORKAREA`。
- 新增 `worker_rest_calendar/desktop_display_configuration` MethodChannel。
- 显示变化后防抖 350ms，再读取稳定后的显示器工作区并恢复摆件。
- 本地物理显示器和远程虚拟显示器按 ID 分别恢复；远程显示器没有独立记录时沿用当前语义位置，断开后恢复本地记录。
- 完整主窗口移动不会写入桌面摆件位置。

### 4. 版本与构建

- 版本升级为 `0.1.9+10`。
- Windows Release：`build/windows/x64/runner/Release/worker_rest_calendar.exe`。
- 安装包：`build/installer/工作日历-Setup-0.1.9.exe`。
- Changelog、UI 规范、测试验收和 Windows 发布记录已同步。

## 代表性代码前后对比

### 旧行为

```dart
final position = await windowManager.getPosition();
await positionStore.save(display.id, position);
await windowManager.setPosition(stored.position!);
```

只保存绝对坐标，分辨率变化后相对屏幕位置漂移。

### 新行为

```dart
final placement = DesktopWindowPlacement.capture(
  position: DesktopWindowPosition(position.dx, position.dy),
  windowSize: DesktopWindowSize(windowSize.width, windowSize.height),
  workArea: currentWorkArea,
);
await positionStore.savePlacement(display.id, placement);

final resolved = placement.resolve(
  windowSize: currentWindowSize,
  workArea: currentWorkArea,
);
await windowManager.setPosition(Offset(resolved.x, resolved.y));
```

按当前显示环境重新计算对应位置。
