# 017 首页剩余工作日计数修复完成

## 任务目标

修复完整主窗口中“再上 X 天班”误把距离下次休息日的日历天数当作工作日数的问题。用户确认真实场景为大小周中的“大周”：2026-07-14 周二至 2026-07-18 周六距离 4 个日历天，但只剩周三、周四、周五 3 个实际工作日。

## 已完成内容

1. `TodayDashboard` 新增 `remainingWorkDaysBeforeNextRest`，从明天起统计到下次休息日之前。
2. 统计基于 `effectiveKind.isActualWork`，因此会正确遵循手动覆盖、调休上班、调休休息和请假。
3. `NextRestCountdownCard` 保留 `daysToNextRest` 判断“今天就是休息日”，其他情况改用实际剩余工作日展示。
4. 移动布局和桌面宽布局的两个卡片入口已同步接入新数据。
5. 新增 3 项领域回归测试和 1 项页面文案回归测试，精确覆盖大小周中当前大周。
6. 版本更新为 `0.1.10+11`，Windows 安装器更新为 `0.1.10`。

## 代码修改前后对比

### 修改前

```dart
final countdown = daysToNextRest == 0
    ? '今天就是休息日'
    : daysToNextRest == 1
    ? '再上 1 天班'
    : '再上 $daysToNextRest 天班';
```

### 修改后

```dart
final remainingWorkDaysBeforeNextRest = daysToRest <= 1
    ? 0
    : schedule
          .days(today.addDays(1), daysToRest - 1)
          .where((day) => day.effectiveKind.isActualWork)
          .length;

final countdown = daysToNextRest == 0
    ? '今天就是休息日'
    : remainingWorkDays == 1
    ? '再上 1 天班'
    : '再上 $remainingWorkDays 天班';
```
