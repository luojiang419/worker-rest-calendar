# 019 安装进度百分比云端发布前

## 任务目标

解决 Windows 自动更新安装阶段只有不确定性循环动画、用户无法判断安装进度的问题；在独立安装窗口中展示真实进度条和 0–100% 百分比，完成回归后推送 GitHub Actions 云端编译并自动正式发布。

## 需求理解

- 已有版本检测、下载、SHA-256 校验、pending、独立 helper、静默覆盖和新版重启链路保持不变。
- 本次只改善 helper 启动安装器后的用户可见反馈，不修改更新策略、网络模式、Release API、资产名称或版本递增契约。
- 百分比必须来自安装器真实工作量，不使用计时器模拟或固定速度增长。
- 安装中继续禁止取消，避免半安装；失败仍需保留 helper 日志和 Inno 安装日志。

## 最终设计

初步评估过“自定义 Win32 进度窗口 + 进度文件 IPC”，但实现需要约 400 行原生窗口、线程和父子进程同步代码。核对 Inno Setup 官方能力后确认：

1. `/SILENT` 本身会保留独立安装进度窗口，只有 `/VERYSILENT` 会将其隐藏。
2. `CurInstallProgressChanged(CurProgress, MaxProgress)` 直接提供解压文件、创建快捷方式、INI 和注册表等真实安装进度。
3. 因此采用更小、更可靠的方案：helper 只把 `/VERYSILENT` 改为 `/SILENT`；Inno 脚本在原生进度窗口标题和状态文本中同步显示百分比。

该方案无文件轮询、无 IPC、无第二套进度条，也不会出现父窗口百分比与安装器真实进度不同步。

## 已完成内容

1. 独立 helper 改用 `/SILENT` 启动 Inno Setup，显示标准 Windows 安装进度窗口。
2. 安装器窗口标题显示 `工作日历更新 - {percent}%`。
3. 安装器页面显示目标版本 `正在更新到工作日历 {version}`。
4. 状态文本显示当前安装动作和百分比，Inno 原有文件标签继续显示正在处理的文件。
5. 百分比通过 `CurInstallProgressChanged` 计算并限制为 0–100；`ssPostInstall` 明确显示 100%。
6. `AllowCancelDuringInstall=no` 与 helper 的 `/NOCANCEL` 共同禁止安装中断。
7. 非零安装器退出码继续由 helper 显示失败提示，原有 updater 与 installer 日志路径不变。
8. 更新 `CHANGELOG.md` 和自动更新发布契约，明确自更新引导边界。


### 安装器真实百分比

新增：

```pascal
procedure CurInstallProgressChanged(CurProgress, MaxProgress: Integer);
var
  Percent: Integer;
begin
  if MaxProgress <= 0 then
    Exit;
  Percent := Round((CurProgress * 100.0) / MaxProgress);
  if Percent < 0 then
    Percent := 0
  else if Percent > 100 then
    Percent := 100;
  SetInstallProgress(Percent, '正在安装和配置…');
end;
```

## 自更新引导边界

当前已安装的旧 helper 仍硬编码 `/VERYSILENT`。因此用户首次升级到包含本改动的版本时仍看不到新进度窗口；安装该版本后，下一次更新由新 helper 传入 `/SILENT`，百分比窗口才会生效。若要做真实 UI 端到端验收，需要先安装首个含本改动的云端版本，再发布更高补丁版本进行更新测试。
