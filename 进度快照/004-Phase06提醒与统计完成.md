# 进度快照 004 · Phase 06 提醒与统计完成

记录时间：2026-07-13（Asia/Shanghai）

## 已完成内容

- 纯 Dart `ReminderScheduler` 输入未来60天有效排班，生成明日状态、调休上班、周预览、休息倒计时四类计划。
- 通知 ID 使用“类型前缀 + YYYYMMDD”稳定推导，重复计算结果一致且四类互不冲突。
- 调休上班使用设置中的提前0–30天；调休和倒计时复用明日状态时间，符合既有数据库 schema。
- 通知 payload 使用 `restcalendar://date/YYYY-MM-DD`，前台点击与冷启动点击均解析到日期详情。
- 平台通知能力封装在 `ReminderPlatformAdapter`，业务层不直接依赖插件。
- 接入 `flutter_local_notifications 22.0.1` 与 `flutter_timezone 5.1.0`，使用真实IANA时区构造 `TZDateTime`；平台时区失败时才回退到系统当前固定偏移。
- Android加入重启恢复权限与两个计划通知 receiver，使用 `inexactAllowWhileIdle`，不申请精确闹钟权限。
- iOS初始化 `UNUserNotificationCenter` delegate；Darwin一次最多提交最早64条计划。
- Windows使用稳定 AppUserModelId 与 GUID 初始化，平台生成注册文件已包含时区插件。
- `ReminderController` 监听 Phase 05 的 `ScheduleRefreshEvent`，排班保存/删除后自动整体替换旧通知。
- 提醒页面处理已授权、未授权/系统关闭、不支持三种状态，支持授权和返回后刷新状态。
- 实现月度六类统计：计划工作、实际工作、休息、请假、调休上班、调休休息。
- 实现截至今天及月内最长的计划/实际连续工作天数，请假只中断实际连续工作。
- 实现365/366天年度状态热力图、月份/年份切换和“统计”底部导航。
- 新增15项测试，全量由75项增加至90项。

## 关键代码前后对比

### 通知计划

修改前：只有提醒设置数据表，没有通知计划模型或系统调度。

修改后：

```dart
final plans = ReminderScheduler(schedule.engine).build(
  preferences: preferences,
  nowLocal: localNow,
  horizonDays: 60,
);
await adapter.replaceAll(plans, timeZoneId: preferences.timeZoneId);
```

通知计划先在纯 Dart 中确定，再由平台 adapter 取消本应用旧 ID 并重建。

### 排班变化后重排

修改前：`ScheduleRefreshEvent` 已发布，但没有消费者。

修改后：

```dart
ref.listen(scheduleRefreshEventProvider, (previous, next) {
  if (next != null && previous?.sequence != next.sequence) {
    unawaited(reschedule());
  }
});
```

日期覆盖保存或删除后，提醒控制器读取最新共享排班并替换所有相关计划。

### 时区

修改前：`ReminderPreferences.timeZoneId` 默认为 `local`，但 timezone 包的默认 local 实际为 UTC。

修改后：平台 adapter 通过 `FlutterTimezone.getLocalTimezone().identifier` 获取设备IANA时区，初始化2025c时区数据库后构造目标本地时间，支持夏令时规则。

### 统计口径

修改前：首页只有简化的工作/休息/调休/请假数量。

修改后：

```text
计划工作 = plannedKind == work
实际工作 = effectiveKind 为 work 或 adjustedWork
休息 = effectiveKind 为 rest 或 adjustedRest
请假 = effectiveKind == leave
调休上班/调休休息另行独立计数
```

月内最长与截至今天的连续工作均分别按计划/实际口径计算。

## 尚存风险或平台限制

- iOS/macOS需在macOS构建机补验；当前仅完成代码、清单和生成注册文件检查。
- Windows通知插件没有与Android/Darwin等价的授权状态查询接口，应用视为平台可用；用户在Windows系统层静默关闭通知时，计划仍会保存但系统可能不展示。
- Android使用非精确省电调度，通知可能由系统做小幅延后，但无需敏感的精确闹钟权限。
- iOS/macOS系统仅保留有限数量的待处理通知，本实现提交按时间排序的最早64条，并在每次启动/排班变化时滚动补齐。
- “统计”已加入导航；“我的”设置总页尚未实现，提醒设置暂由今日页铃铛入口进入。
