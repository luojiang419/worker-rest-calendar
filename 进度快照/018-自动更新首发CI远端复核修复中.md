## 当前修改模块

当前处于“首个自动更新基线版本 v0.1.11 云端发布”模块。前三次 Actions 已验证编译、测试、安装包和校验文件都可成功生成；第三次仅因 GitHub 草稿 Release 不能通过 `/releases/tags/{tag}` 查询而在发布前远端复核失败，草稿已按 Release ID 手动清理，未留下标签或公开 Release。

## 代码修改前后对比

### 修改前

```powershell
$release = gh api "repos/$env:GITHUB_REPOSITORY/releases/tags/$env:TAG" | ConvertFrom-Json
gh release download $env:TAG --pattern "$env:ASSET_NAME.sha256"
gh release edit "$env:TAG" --draft=false --latest
```

草稿 Release 对 tag 查询接口返回 404，导致远端复核无法继续。

### 修改后

```powershell
$matches = @(gh api "repos/$env:GITHUB_REPOSITORY/releases?per_page=100" |
  ConvertFrom-Json | Where-Object {
    $_.tag_name -eq $env:TAG -and
    $_.draft -and
    $_.target_commitish -eq $env:GITHUB_SHA
  })
"RELEASE_ID=$($matches[0].id)" | Out-File $env:GITHUB_ENV -Append

$release = gh api "repos/$env:GITHUB_REPOSITORY/releases/$env:RELEASE_ID" |
  ConvertFrom-Json
gh api -X PATCH \
  "repos/$env:GITHUB_REPOSITORY/releases/$env:RELEASE_ID" \
  -F draft=false -f make_latest=true
```

远端校验和下载也改为通过 Release asset ID 获取；失败清理只删除本次 `tag + draft + target_commitish` 精确匹配的草稿，并只在远端标签恰好指向本次提交时删除该标签。
