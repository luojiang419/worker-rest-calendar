## 用户最新反馈与根因

用户截图显示桌面摆件外侧存在一圈矩形深色区域。

根因已经确认：

- `DesktopWidgetCard` 上一轮增加了 `Padding(padding: EdgeInsets.all(8))`，用于给外部投影留下安全边距。
- `WorkerRestCalendarApp` 的 `_ThemeBackdrop` 会给整个 Flutter 根窗口铺主题背景色。
- 因此本应透明的 8px 安全边距在暗色主题下变成深色矩形边框。
- 卡片自身还存在风格化的细边界，这是另一层，不是截图中主要的矩形黑框。

用户已确认推荐方案：

- 保留 8px 阴影安全边距，但必须使其真正透明。
- 经典、玻璃、纸感只保留极淡高光边缘；拟物无边框；扁平保留中性细边界。
- 所有风格不得出现深色外围描边。

## 当前代码与目标对比

### 摆件背景当前行为

```dart
builder: (context, child) => _ThemeBackdrop(child: child!),
```

`_ThemeBackdrop` 对非玻璃风格使用：

```dart
return ColoredBox(color: tokens.colors.background, child: child);
```

目标行为：完整主窗口仍使用主题背景/玻璃渐变；桌面摆件模式下根窗口背景必须透明，只绘制卡片及其阴影。

### Windows 当前没有单实例锁

`windows/runner/main.cpp` 每次启动都会直接创建新的 `FlutterWindow`，没有命名互斥量、激活消息或参数转发。

目标行为：

- Windows 当前用户会话内只允许一个实例。
- 第二次启动不创建 Flutter 引擎和数据库连接。
- 若现有实例处于摆件模式，切换为完整主窗口。
- 若主窗口已打开、被遮挡或最小化，恢复并置于前台。
- 转发命令行、通知或日期激活参数，避免单实例锁破坏通知点击。

推荐实现：Windows 命名 Mutex + 注册窗口消息/`WM_COPYDATA` + Flutter MethodChannel adapter；早期激活请求需要有 pending/consume 机制，不能因 Dart handler 尚未注册而丢失。

### 托盘当前行为

- `TrayManagerDesktopTrayService.onTrayIconMouseDown()` 当前单击就调用打开完整应用。
- `DesktopWidgetController.openFullApp()` 在 `showFullApp == true` 时直接返回，无法让已有主窗口重新前置。
- `openFullApp()` 会隐藏托盘；完整主窗口模式没有托盘图标。
- `WindowManagerDesktopWindowService.showFullApp()` 会解除 bottom/top，但没有完整执行 show、restore、focus。

目标行为：

- 应用进程运行期间托盘图标始终显示，包括主窗口和摆件模式。
- 左键单击不操作；左键双击打开/恢复主窗口并置于前台；右键打开菜单。
- 主窗口已打开时再次激活也要执行恢复和聚焦，不能直接 return。
- 主窗口解除桌面底层和置顶，作为普通任务栏窗口显示在前台。
- 摆件锁定时进入桌面底层；未锁定时保留普通/置顶选项。

### 托盘菜单已确认内容

- 打开/置前主窗口。
- 快捷打开：今天、日历、统计。
- 提醒设置。
- 数据与同步。
- 显示桌面摆件。
- 摆件尺寸、显示层级、透明度。
- 外观模式、视觉风格。
- 开机启动。
- 退出。

主窗口右上角关闭按钮已确认改为：返回桌面摆件并继续驻留托盘；只有托盘“退出”才真正结束进程。

## 已确认的全部产品决策

1. 8px 安全边距保持，但必须真正透明。
2. 不出现深色卡片描边，各风格只保留必要的浅边界或无边界。
3. 单实例锁范围为 Windows 当前用户会话。
4. 重复启动切换/恢复现有主窗口并置于前台。
5. 第二实例激活参数需要转发。
6. 托盘图标在应用整个生命周期持续显示。
7. 托盘单击无动作、双击打开主窗口、右击打开菜单。
8. 托盘菜单采用上文完整快捷功能清单。
9. 主窗口是普通前台窗口；只有摆件可进入桌面底层。
10. 关闭主窗口返回摆件；退出必须通过托盘“退出”。
