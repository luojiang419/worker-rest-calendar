# 进度快照 014 · 多风格主题选择与持久化完成

## 本轮目标

将完整应用原有的浅色/暗黑快速切换按钮改造为主题选择弹层，保留跟随系统/浅色/暗色外观模式，并新增多种独立视觉风格。

## 已完成内容

### 1. 主题领域模型

- 新增 `AppVisualStyle`：`classic`、`flat`、`neumorphic`、`glass`、`paper`。
- 外观模式 `AppThemePreference` 与视觉风格分离，可以自由组合。
- `AppTokens.resolve(style, brightness)` 统一生成 5 种风格的浅色/暗色令牌。
- 所有风格保持工作、休息、调休上班、调休休息、请假的原有语义。

### 2. 材质适配

- 经典精致：保留原有柔和投影和苹果软件气质。
- 现代扁平：移除常规投影，使用色块、紧凑圆角和清晰边界。
- 柔和拟物：浅色与暗色分别配置双向高光/暗影，边界仍依靠文字和状态标签表达。
- 通透玻璃：根背景渐变、半透明表面、边缘高光和受控 `BackdropFilter`；模糊仅使用于卡片/弹层材质。
- 静谧纸感：暖色低饱和表面、短柔阴影与更小圆角。

### 3. 主题选择交互

- 桌面左侧栏底部原明暗按钮改为“选择主题”调色板按钮。
- 移动端“今天”页标题区新增同一主题入口。
- 弹层上方选择跟随/浅色/暗色，下方通过可视化小卡选择 5 种风格。
- 选择后立即更新 Riverpod 主题状态并写入数据库。
- 应用启动时通过一次性 Future provider 读取主题偏好，不保留长期数据库流订阅。

### 4. 持久化与兼容

- Drift 数据库 schema 从 v3 升级为 v4，`app_settings` 新增 `visual_style`，默认 `classic`。
- v1 直升 v4 与 v2 升 v4 迁移测试通过，不会重复添加字段。
- 导出备份新增 `visualStyle`。
- 旧备份缺少 `visualStyle` 时自动使用 `classic`，不提升备份 schema 版本，保持向后兼容。

## 核心代码前后对比

### 原交互

```dart
final isDark = Theme.of(context).brightness == Brightness.dark;
IconButton.filledTonal(
  tooltip: isDark ? '切换到浅色模式' : '切换到暗黑模式',
  onPressed: onToggleTheme,
  icon: Icon(isDark ? Icons.light_mode_outlined : Icons.dark_mode_outlined),
);
```

### 新交互

```dart
IconButton.filledTonal(
  tooltip: '选择主题',
  onPressed: onSelectTheme,
  icon: const Icon(Icons.palette_outlined),
);
```

### 原主题配置

```dart
theme: AppTheme.light,
darkTheme: AppTheme.dark,
themeMode: ref.watch(themeModeProvider),
```

### 新主题配置

```dart
final visualStyle = ref.watch(visualStyleProvider);
theme: AppTheme.lightFor(visualStyle),
darkTheme: AppTheme.darkFor(visualStyle),
themeMode: ref.watch(themeModeProvider),
```
