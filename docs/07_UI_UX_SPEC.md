# 07 · UI/UX 视觉规范

## 1. 风格

苹果软件气质的现代扁平化：克制、留白充足、层级清晰。不是纯平面无层次，按钮、卡片、弹层使用柔和而精致的投影和边缘高光。

## 2. 主题

主题由两个独立维度组成：

- 外观模式：跟随系统、浅色、暗黑。
- 视觉风格：经典精致、现代扁平、柔和拟物、通透玻璃、静谧纸感。

所有视觉风格必须同时提供浅色与暗黑令牌，并保持工作、休息、调休上班、调休休息、请假的语义不变。玻璃风格只在卡片和弹层使用受控模糊，必须提供实色降级表面，不叠加多层模糊。拟物风格通过双向柔光投影表达轻微浮雕，不得以低对比度代替边界与状态文字。

### 浅色

- 页面背景：暖灰白，而非纯白。
- 卡片：白色或轻微半透明白。
- 主文字：近黑色。
- 投影：冷灰低透明度，配合 1px 浅边框。

### 暗黑

- 页面背景：炭黑或蓝黑，不使用纯黑铺满。
- 卡片：比背景略亮。
- 投影：更深黑色外阴影 + 极弱白色边缘描边。
- 状态色适当提高亮度，但控制饱和度。

## 3. 状态颜色

- 工作：中性蓝灰。
- 休息：薄荷绿。
- 调休上班：暖橙。
- 调休休息：青绿色，与正常休息通过标签区分。
- 请假：淡紫。
- 错误/删除：柔和红。

颜色值以 `config/design_tokens.json` 为唯一事实来源。

## 4. 投影层级

- `shadow.low`：日期格、轻量标签、次级按钮。
- `shadow.medium`：普通卡片、分段控件、底部导航。
- `shadow.high`：底部弹层、菜单、桌面挂件悬浮状态。

阴影必须随主题变化。暗黑模式不能仅复用浅色灰影，否则会显脏。

## 5. 圆角与间距

- 小控件：12px。
- 按钮、输入框：14–16px。
- 普通卡片：20px。
- 核心状态卡：24–28px。
- 底部弹层：顶部 28px。
- 页面横向安全边距：16px 手机，24px 平板/桌面。

## 6. 控件

### 主按钮

- 高度至少 48px。
- 蓝色填充，白字，medium shadow。
- 按下时缩放至 0.98，阴影减弱。

### 次按钮

- 半透明表面 + 1px 描边 + low shadow。

### 分段控件

- 月/周切换；选中项有独立浮起的胶囊背景和轻投影。

### 月历浏览

- 月视图使用按需构建的双向自由滚动视口，设置页允许选择左右或上下方向；手指、鼠标、触控笔、鼠标滚轮和触控板均可操作。
- 月历不得整页吸附或在边界回弹，滚动可停在任意像素位置，包括同时显示相邻两个月；视口中心月份使用正常明度，相邻月份降至 58% 明度。
- 视口中心跨过月份中线后，月份标题和选中月份同步更新；跨年浏览不得中断。
- 上一月、下一月和“回到今天”继续保留，按钮切换使用页面状态切换时长和减速曲线。
- 相邻月份只在进入视口时计算日期状态，不一次性生成长期日期数据。

### 设置入口

- 桌面完整主窗口左侧栏底部依次放置主题与设置图标，设置图标位于主题图标下方并保持至少 44×44 点击区域。
- 设置页集中管理开机自启、月历滚动方向、桌面摆件尺寸与表现，并提供外观主题、提醒、数据与同步入口。

### 开关

- 使用平台熟悉的胶囊造型，但颜色遵守产品主题。

### 状态标签

- 胶囊标签，文字与背景均满足可读性。
- 不能只依赖颜色，必须显示“休”“班”“调”“假”等短字或图标。

## 7. 动效

- 快速反馈：120–180ms。
- 页面状态切换：220–280ms。
- 弹层：280–360ms，使用减速曲线。
- 数字倒计时变化可轻微淡入和位移，禁止夸张弹跳。

## 8. 可访问性

- 点击区域至少 44x44。
- 支持系统字体放大到 130% 时不截断核心信息。
- 所有状态有语义标签。
- 暗黑和浅色模式对比度均通过基础检查。
- “工作/休息”不能只靠红绿色区分。

## 9. 桌面摆件主题

- 桌面摆件与完整应用共享同一外观模式和视觉风格，不提供独立主题副本。
- 跟随系统时只自动切换浅色/暗黑令牌，保留用户选择的视觉风格，并在系统外观变化后实时更新。
- 小、中、大三种摆件都必须覆盖五种视觉风格的浅色与暗黑状态；外层保留透明安全边距，避免卡片投影被窗口边缘直接裁断。
- 摆件最外层卡片统一使用无描边圆角表面，只以材质和投影建立层级；Windows 宿主窗口必须完全无边框，不得出现圆角高光线、中性细边界或直角系统外框。
- 中号七日格和大号月历日期格随风格改变材质。柔和拟物下普通日期轻微浮起，今天使用主色方向渐变表达按下层次，悬停与按下使用快速反馈时长。
- 中号七日格与大号月历默认使用同一套主题自适应动态微光描边突出今天，描边模式不得改变原有日期底色和文字颜色；用户可切换为填充高亮。大号月历另允许在圆角矩形与圆形日期格之间切换，所有选择均持久化。
- 小号摆件的今日状态与倒计时使用同风格表面。所有尺寸继续显示文字和状态色，立体效果不得替代“班、休、调、假”等语义。
- 右键菜单使用操作系统原生菜单，在当前鼠标位置弹出并由系统限制到当前显示器可用区域；菜单外观跟随操作系统，不模拟应用的五种视觉风格。
- 原生菜单按尺寸、大号日期样式、当日突出样式、显示层级、透明度、外观模式和视觉风格分组，避免在小号摆件窗口内出现滚动菜单。
- Windows 应用在当前用户会话内保持单实例；重复启动、托盘双击和外部激活均恢复并前置现有完整主窗口，不创建第二个 Flutter 引擎。
- 托盘图标在完整应用和桌面摆件模式下都持续显示。左键单击不执行操作，左键双击打开或恢复主窗口，右键打开原生菜单。
- 托盘菜单提供主窗口前置、今天、日历、统计、提醒设置、数据与同步、显示桌面摆件，以及摆件和主题设置；只有托盘“退出”结束进程。
- 托盘“退出”必须立即隐藏可见窗口，窗口销毁与托盘清理并行执行；连续触发必须幂等，不得因原生资源清理延迟造成界面卡顿。
- Windows 主窗口使用普通任务栏层级；关闭按钮返回桌面摆件并继续驻留托盘，锁定摆件才进入桌面底层。
- 桌面摆件位置以显示器可用工作区为基准持久化：距离边缘不超过定位阈值时保持对应边距，中间区域保持可移动范围内的相对比例；分辨率、DPI、任务栏工作区和远程虚拟显示器变化后必须重新换算并保证窗口不越界。主窗口位置不得覆盖摆件位置记录。
