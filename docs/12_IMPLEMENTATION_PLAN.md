# 实施路线图

本项目严格按 `codex/00` 至 `codex/10` 顺序实施。每次只推进一个 Phase，达到退出条件后才更新 `PROGRESS.md`。

| Phase | 模块 | 核心验收 |
| --- | --- | --- |
| 00 | 工程底座 | 四端目录、依赖锁、主题入口、CI、基础测试 |
| 01 | 排班领域 | 纯 Dart 规则、覆盖优先级、边界与统计测试 |
| 02 | 本地数据 | Drift schema、仓储、迁移、事务、导入解析 |
| 03 | 设计系统 | 令牌、双主题、统一组件、可访问性测试 |
| 04 | 首次设置 | 班制选择、锚点、自定义循环、预览与恢复 |
| 05 | 日常主流程 | 今日、月/周日历、详情与单日编辑一致性 |
| 06 | 提醒统计 | 稳定通知调度、深链、月/年统计 |
| 07 | 桌面挂件 | Windows/macOS 三尺寸及窗口能力适配 |
| 08 | 手机小组件 | iOS/Android 原生三尺寸与共享快照 |
| 09 | 数据迁移 | 导出、导入预览/回滚；云同步受开关控制 |
| 10 | QA 发布 | 全量测试、四端构建、隐私与发布材料 |

## 全局验收纪律

每个 Phase 完成时运行格式化、静态分析、单元测试和已有集成测试；失败时不勾选进度。领域定义、覆盖优先级和 date-only 存储规则不得改变。
