# 01 · 产品需求文档 PRD

## 1. 背景

双休、单休、大小周、轮班和临时调休经常交叠，普通日历不会直接回答“今天上不上班”和“下次什么时候休息”。本产品以个人休息节奏为核心，而不是以会议或待办为核心。

## 2. 目标用户

- 固定双休或单休的上班族。
- 大小周交替工作的人群。
- 做六休一、做二休二等循环班制用户。
- 每月排班或临时换班较多的人群。

## 3. 核心任务

用户在 3 秒内完成以下判断：

- 今天是工作日、休息日、调休上班还是调休休息。
- 距离下一休息日还有几天。
- 当前周是大周还是小周。
- 某个未来日期是否需要上班。

## 4. MVP 范围

### 必须实现

- 双休、单休、大小周、做六休一、做二休二、自定义循环。
- 大小周锚点设置和实时预览。
- 月视图、周视图、今日首页。
- 手动修改单日状态；添加请假、加班时长和备注。
- 浅色、暗黑、跟随系统三种主题模式。
- 本地提醒：明日状态、调休提醒、下次休息倒计时、周预览。
- 月度统计与连续工作天数。
- Windows/macOS 桌面挂件，小/中/大尺寸。
- iOS/Android 主屏小组件，小/中/大尺寸。
- 本地数据导出与导入。

### 后续可选

- 官方节假日数据包在线更新。
- Supabase 跨设备同步。
- 团队排班共享。
- 工资与考勤计算。

## 5. 非目标

- 不替代企业考勤系统。
- 不处理复杂薪资、税务或工时合规计算。
- 不要求用户注册后才能使用。
- 不把普通日历会议管理作为主功能。

## 6. 成功指标

- 首次设置在 90 秒内完成。
- 用户查看“今天是否上班”的路径不超过一次点击。
- 大小周未来 12 个月计算准确率为 100%。
- 手动调整后，手机、桌面挂件与通知状态保持一致。
- 冷启动后首页核心状态在 500ms 内从本地缓存呈现；允许后台再刷新。

## 7. 关键文案原则

- 直接：使用“今天上班”“今天休息”，不用晦涩术语。
- 轻松但不油腻，例如“再上 2 天班”。
- 关键提醒必须明确日期和星期，避免只写“明天”。
